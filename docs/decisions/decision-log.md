### ADR-001: Pipeline orchestration substrate: per-pipeline prep|gate|finish scripts, Workflow deferred

**Date**: 2026-07-07 | **Session**: 17 | **Status**: Active

**Context**: The catalog/audit/enrich pipelines ran their deterministic control flow (arg parse, sharding, gating, dedup) as bash prose across SKILL.md Bash blocks, but the harness wipes shell env between blocks, so state silently evaporated (#72); the gates proved a file was written, not that every dispatched item was processed (#71); and the fan-out was hand-copied per skill (#76).

**Decision**: One checked-in orchestration script per pipeline (`scripts/pipeline/{catalog,audit,enrich}.sh`) with `prep|gate|finish` phase subcommands, mirroring `ingest-book` Step 3B. State crosses phases through files under `dev/<phase>/` (`run.env`, `dispatch.tsv`), never shell env. Model fan-out stays skill prose. A shared `check-coverage.sh` reconciles dispatched ids against per-item receipt rows. One `references/article-contract.md` SSOT replaces five drifted schema restatements. Full design in `dev/specs/pipeline-orchestration-ssot.md`; 10-task plan in `dev/plans/pipeline-orchestration-ssot.md`.

**Alternatives**: Workflow-tool-only (rejected — a workflow script has no filesystem/Bash access, so gates/dedup/staging/commits can't live there; confirmed first-party). Migrating fan-out to the Workflow tool was measured head-to-head against parallel-`Agent` dispatch on a live 15-gap enrich run (T6, Session 18): substrate-neutral on tokens/wall-clock/yield, its schema-robustness and context-isolation wins already gate-backstopped and only binding at ~100-item fan-out — so Workflow is deferred and Agent-calls retained for all three pipelines (T9 resolved, no code; record in `dev/t6-measurement/decision.md`). Python orchestrators (rejected — every existing helper is bash).

**Consequences**: All three pipelines shipped (enrich 0.46.0, audit 0.47.0, catalog 0.48.0). `check-coverage.sh` hardened after a codex review found three false-pass holes (0.45.1). A per-batch reconciliation gap remains (#92). Epic #87 closed Session 18 with #72/#71/#76; #77 stays open for its downstream schema sub-issues (#88-#91, of which #91 was closed as misfiled).

### ADR-002: Worker stages use a cheap-tier verify-and-fix recipe; harness owns meta-judgments; determinism retired

**Date**: 2026-07-12 | **Session**: 25 | **Status**: Active

**Context**: Every worker stage (synthesis, validation, extraction, enrichment) ran the authoritative cloud model (sonnet) to do the work *from scratch*, spending Claude subscription quota on bulk generation. S25 measurement showed a cheap tier clears each stage's accuracy floors **behind the harness**: local qwen3-30b-a3b (~$0, off-quota) and — measured the same way — subscription haiku both clear the synthesis (recall + 0 over-claims) and validation (poison recall 1.00 / false-correction 0) floors. The failures the cheap tier does make are all *meta-judgments* (refuse-or-write, tag conformance, dedup, over-mint), not the content itself.

**Decision**: Worker stages use a **verify-and-fix** recipe. (1) The cheap tier does exactly **one object judgment** per call (draft the article / judge the claim / extract concepts / ground the gap), serially on the single local GPU (a3b is fast — ~197 tok/s, ~65s per 20-slug batch). (2) A **deterministic harness gate owns every meta-judgment** (refusal gate, tag-vocab filter, citation normalizer, slug pre-match, URL-dedup set-membership) — code, not the model. (3) The authoritative cloud model **verifies** the cheap output against the stage floors with a specific, free-form critique and **fixes only what fails** (≤2 rounds), rather than regenerating. The verifier must differ from the drafter family (qwen-draft + sonnet-verify is clean; haiku-draft shares the Claude family). The goal is **good output that clears the floors, not byte-identical output** — determinism (temp-0 byte-identity) is **retired as a criterion**; it was a testing scaffold to prove the harness. Rollout is advisory-first (grade the drafts before flipping authoritative), synthesis first, then validation/extraction/enrichment on the identical recipe.

**Alternatives**: Cloud-generates-from-scratch (status quo — rejected: spends quota on work the cheap tier does well behind the harness). Byte-determinism as a gate/differentiator (rejected: good ≠ identical; it was a scaffold, and the meta-judgment failures reproduce deterministically at temp 0 so stabilizing sampling can't fix them). Local-only, no cloud verify (rejected: floors were measured on small offline sets, so cloud verify is the safety net and preserves the cloud-authoritative contract; and the generator-verifier gap means a weak drafter's coarse errors are the *easy* case for a verifier — 75.7% gap compression under a shared verifier). Fixing meta-judgments *inside* the model via better prompts / calibration / majority-vote (rejected: generator-verifier-gap + structure-snowballing + the S25 refusal/dedup prompt-fragility show these are model-intrinsic, not promptable).

**Consequences**: Advisory verify window shipped (0.62.0, hardened 0.62.1 after a codex review caught five edge cases). Synthesis is slice 1; validation/extraction/enrichment follow the same recipe. Durable design principle + per-stage table: `dev/experiments/model-tier/DESIGN-local-tier.md`. Rollout plan + file-flow + acceptance criteria: `dev/specs/verify-and-fix.md`. Grounding: the LLM knowledge stack (generator-verifier-gap, self-correction-loops, llm-judge-gate-wiring) and the S25 measurements (`dev/experiments/model-tier/results-*.md`). Throughput measured on the local box (liminal): keep_alive:-1, num_ctx 4096, /api/chat. The local agentic enrichment loop (search+fetch+judge) is proven, so all four stages fit the recipe.
